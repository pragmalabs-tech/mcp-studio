# 0.3.31

### Features

**Fullscreen display mode**
- Widgets can now request fullscreen display via the `ui/request-display-mode` JSON-RPC method. The studio honors the request, switches the shell layout, and sends `ui/notifications/host-context-changed` back to the widget.
- A new fullscreen shell wraps the widget content in an expanded view with the chat input bar overlaid at the bottom.
- An **✕ button** in the widget app header lets users exit fullscreen and return to inline mode at any time.

**Viewport presets**
- The preview bar now renders the widget at the correct dimensions for **Desktop**, **Tablet**, and **Mobile** presets, matching common real-world host screen sizes.

**Display mode rendered inside the widget**
- The widget frame reflects the active display mode (inline / fullscreen) so widgets can adapt their own layout accordingly.

### Bug fixes

- **Widget no longer re-renders on display mode / theme / locale changes.** Previously, changing any mock setting (theme, locale, display mode) destroyed and recreated the iframe, resetting widget state. The mock is now updated in-place via `update()` without re-writing HTML.
- **OpenAI ext-apps widgets (e.g. Excalidraw) no longer go blank on display mode change.** The `applyMock` fallback path was resetting `snapshot: null` for widgets using the JSON-RPC protocol instead of legacy `window.openai` globals, causing a full re-render. It now calls `update()` directly when an ext-apps mock is active.
- **Switching display modes no longer unmounts the widget iframe.** The inline and fullscreen shells previously lived in separate JSX branches, causing React to unmount and remount `WidgetRenderer` on every switch. The shell is now a single component that changes its layout CSS while keeping the iframe at a stable tree position.
- **Mock events now reach the widget after the first render cycle.** The snapshot update (`setSnapshot`) caused the effect that creates the mock to bail out early, leaving no mock registered after the first 150ms. The mock lifecycle is now decoupled from the snapshot guard.
- **Redundant `tool-result` re-send on display mode request eliminated.** When a widget requested a display mode change, `onDisplayModeChange` triggered `applyMock` which called `sendToolData` again — sending `tool-result` a second time and causing widgets like Excalidraw to reset. The callback now writes directly to the store without going through `applyMock`.
