# 0.3.34

### Features

**Canvas widget interaction recording & replay**
- Clicks on `<canvas>` elements (e.g. Excalidraw) are now captured as first-class actions alongside regular HTML clicks. The tap position is stored as normalized coordinates (`nx`/`ny`) so it is independent of canvas size.
- Double-clicks and triple-clicks on canvases are recorded correctly — successive clicks on the same canvas within the settle window are folded into a single action with the highest `detail` count, matching browser behaviour.
- Text editing inside canvas widgets (e.g. Excalidraw's inline text editor) is captured: a following text-input step re-opens the ephemeral editor before typing so the keystroke lands in the right place.

**Viewport size lock during replay**
- The iframe viewport is locked to the exact pixel dimensions recorded at capture time (`window.innerWidth/innerHeight` inside the widget frame). Canvas libraries map screen pixels to scene coordinates at a fixed zoom, so replaying at a different size would land taps in the wrong place. The lock is released automatically once the replay finishes.

**Warning assertion mode**
- A new `warn` mode is available for assertion points. Like `exact`, it compares expected vs actual and shows the diff — but a mismatch never fails the step. The row appears with a yellow indicator instead of red.
- **Canvas found** and **Error message** on canvas click steps default to `warn` because canvas selectors are positional and not stable across re-renders. The position (`nx`/`ny`) is what matters; the selector is informational.

### Improvements

- **Canvas click step label**: position is now shown as `x0.67 y0.37` (normalized 0–1 scale) instead of `67%×37%`, making it clearer that these are scene-independent coordinates.
- **Canvas click reliability**: `execute()` now polls for the canvas element for up to 3 seconds before giving up. Previously a single immediate lookup failed when Excalidraw hadn't finished mounting, causing flaky failures in auto-run mode that didn't appear in step-by-step mode.
