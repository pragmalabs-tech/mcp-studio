# 0.3.37

### Improvements

**Paginated history loading**
- The history drawer now loads replay results in pages of 30 instead of fetching the entire run catalog at once. A "Load more" button appears at the bottom when more results are available. This keeps the drawer fast to open regardless of how many runs have accumulated.

**Server-side test filtering**
- The run-results API now accepts `limit`, `offset`, and `test_id` query parameters. Per-test history lookups (used by the test card's history badges) filter on the backend instead of pulling all summaries and discarding unrelated runs.

**History loading indicator**
- A spinner is shown while the history drawer is fetching its first page, replacing the previous behaviour where the drawer appeared empty until the request completed.
