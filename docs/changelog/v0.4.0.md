# 0.4.0

### Features

**Headless test runner**
- Tests can now be run without opening a browser window using the `--headless` flag. Pass one or more `--test-id <id>` arguments to specify which tests to run. MCP Studio launches Chrome in the background, executes the tests, and exits with a non-zero code on failure. This makes it suitable for CI/CD pipelines.

```
mcp-studio --headless --test-id my-test --test-id another-test
```

**Server-triggered test runs**
- The server can now trigger test runs directly over a WebSocket control channel. Each trigger creates a job with a unique ID that can be used to track and retrieve the result. This underpins the headless runner and opens the door for future programmatic integrations.

**Job result API**
- Added an endpoint to fetch the result of a triggered test run by job ID. Poll it after triggering a test to get pass/fail status and output without needing to watch the frontend.

### Improvements

**CLI rewritten with clap**
- The command-line interface now uses [clap](https://docs.rs/clap) for argument parsing. `--help` and `--version` work out of the box, arguments are validated on startup, and error messages are clearer when a required value is missing.
