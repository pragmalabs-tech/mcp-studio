# 0.3.2

### Features

**Excalidraw-ready default profile**
- Fresh installs now ship with the `default` profile pre-pointed at `https://mcp.excalidraw.com/mcp`, so the app has something to talk to out of the box. Existing installs with an empty default URL are back-filled on next launch.
- The Profiles dialog now opens directly into **edit mode** on the active profile — URL, auth, and name fields are immediately reachable instead of hidden behind a pencil icon.
- The Profiles dialog auto-opens on launch when only the seeded `default` profile exists, nudging new users to swap in their own MCP target instead of silently running against the demo server.

**OpenAI single-widget app fallback**
- When the MCP server exposes exactly one `text/html;profile=mcp-app` resource (the Excalidraw pattern — one canvas serving every tool), Studio now uses it as the widget for any tool call that doesn't name a UI resource explicitly. Previously these renders silently dropped to "no widget".

### Bug fixes

- **Logs panel no longer shrinks when executing large tool calls.** The Request Editor textarea had `field-sizing: content`, which made it grow to fit the input and squeeze the logs/console pane below it. The bottom pane is now a fixed-pixel split (default 320px, still drag-resizable) and the textarea is opted out of content sizing, so neither half affects the other.

### Removed

- **Strict CSP toggle** has been removed from the widget config bar. It interfered with widget rendering in several cases. The CSP panel still surfaces violations detected during normal rendering.
