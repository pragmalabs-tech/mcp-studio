# 0.3.3

Hotfix release focused on internal widget stability and reliability.

### Bug Fixes

- **Widget events no longer silently dropped.** Event listeners were being lost on re-renders; the message bus is now stable across the widget lifecycle.
- **HTML Source tab shows original markup.** The HTML Source tab was displaying injected/transformed HTML instead of the original response from the server. It now correctly shows `originalHtml`.

### Refactoring

**Widget injection modularized**
- The monolithic `render-html.ts` and `widget-renderer.tsx` injection logic has been split into focused modules under `lib/core/widget/injections/`:
  - `capture-events.ts` — click and text-input action recording
  - `claude-link-intercept.ts` — intercepts `claude://` protocol links
  - `console-forwarder.ts` — forwards `console.*` calls from the iframe to the Studio console
  - `content-height.ts` — auto-height resize messaging
  - `csp-meta.ts` — CSP `<meta>` tag injection
  - `openai-mock.ts` — legacy `window.openai` mock shim
  - `recorder-bridge.ts` — connects the recorder to widget actions
  - `sandbox-trap.ts` — sandbox escape detection
  - `view-only.ts` — disables interaction in view-only mode
- Each injection is independently testable and opt-in via `inject.ts`.

**Widget message bus extracted**
- `widget-message-bus.ts` centralises all `postMessage` handling that was previously scattered across `WidgetRenderer`. The component is now a thin shell that mounts the iframe; all async message handling lives in the bus.

**Widget snapshot extracted**
- `widget-snapshot.ts` owns snapshot scheduling, replacing ad-hoc `setTimeout` calls inside the renderer.

**Auto-height moved to store**
- `autoHeight` is now reactive state in `widget-store` rather than local `useState` inside `WidgetRenderer`, making it accessible to other consumers without prop drilling.
