# 0.3.1

### Features

**Test tagging**
- Tests can now be labelled with free-form tags. Add tags when saving a new test or inline from the expanded test card. Tags are lowercase, deduplicated, and persisted alongside the test.
- A **Filter by tag** pill bar appears in the Tests panel once any tags exist. Selecting one or more pills narrows the list to tests that carry *all* selected tags (AND semantics). Active filters are reflected in the URL (`?tags=auth,smoke`) so they survive navigation and can be shared.
- The **Run All** button respects the active tag filter — it runs only the currently visible tests.

**Run All**
- New **Run All (N)** button in the Tests panel header. Runs every visible test in sequence against the live MCP server, reusing the existing run bar to show per-test progress (`[2/5] Login flow`). The stop button cancels the entire batch.
- After the batch completes a summary dialog shows each test's pass/fail status and duration. Each row has a **View** button to open the full replay result.

### Bug fixes

- **OAuth token refresh** — after a silent background refresh the auth panel timer no longer stays stuck on "expired". The store's `expiresAt` is now patched whenever `ensureValidToken` refreshes the token automatically before an MCP call.

### Assertion improvements

- **Flaky mode** now treats scalar arrays (`string`, `number`, `boolean`, `null`) as order-insensitive. `["water", "CO2"]` and `["CO2", "water"]` are considered equal under flaky, so tools that return the same set of values in a random order no longer produce false failures.
