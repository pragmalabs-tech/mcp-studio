# 0.3.32

### Features

**Resource HTML preview**
- When `resources/read` returns an HTML response (`text/html` mime type), it is automatically loaded into the widget iframe using the full MCP Apps postMessage protocol — the same rendering path as widget tools.
- Preview and HTML Source tabs appear automatically for HTML resources.
- The Data tab shows the resource result metadata; the large HTML body is replaced with a placeholder to keep it readable, with the full source available in HTML Source.

**Test Data injection for resources**
- Resources with HTML now get a **Test Data** tab in the request editor. Paste custom `toolInput`/`toolOutput` JSON and hit **▶ Inject** to push data directly into the running iframe without re-executing the resource. Useful for developing and testing MCP App widgets with fake data.

**Reload Widget**
- A **↻ Reload Widget** button in the preview panel clears the HTML cache, refetches the latest version from the server, and reloads the iframe while preserving any injected test data. Designed for local development workflows with HMR.

### Improvements

- **Preview tabs**: Preview and HTML Source tabs are now hidden for plain tool calls with no widget — only the Data tab is shown, reducing noise.
- **Copy button**: Fixed copy button not appearing on the Data tab when it was the only visible tab (the tab state was `"preview"` while the effective tab was `"mock"`).
- **Widget detection**: Preview/HTML Source tab visibility now correctly derives from the tool's `ui://` meta URI rather than the widget store cache state.
- **Profile panel**: Collapses by default.
- **URL**: Updated footer link to `studio.pragmalabs.tech`.

### Bug Fixes

- **Install script**: Fixed post-install run command from `mcp-studio open` to `mcp-studio`.
