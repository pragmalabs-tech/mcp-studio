# 0.3.38

### Fixes

**Snapshot capture during tool calls and recording**
- Fixed an issue where snapshots were not correctly captured when calling a tool or recording an action. Removed the snapshot-center approach in favour of direct capture at the point of interaction.

**Widget render timing**
- Added a 1-second delay between each replay step to give widgets enough time to fully render before the next action is captured. Prevents false mismatches caused by partially rendered UI.

### Improvements

**Accurate layout snapshots**
- The runner now captures the bounding box of each element before taking a snapshot, ensuring the recorded layout matches the actual rendered position regardless of scroll or viewport offset.

**Simplified runner internals**
- Consolidated replay runner logic to reduce complexity and make future changes easier to reason about.
