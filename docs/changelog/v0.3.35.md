# 0.3.35

### Improvements

**Accurate canvas snapshots in step review**
- Widget snapshots now capture `<canvas>` pixel content correctly. Previously, canvas elements (e.g. Excalidraw drawings) appeared blank in the replay result dialog because `outerHTML` alone cannot preserve GPU/memory pixel buffers. Snapshots now replace each canvas with an `<img>` tag using `canvas.toDataURL()`, so the visual state at capture time is faithfully preserved.

**Display mode indicator**
- When a widget requests a non-inline display mode (e.g. `sidebar`, `fullscreen`), a small amber badge now appears in the widget header showing the active mode. Makes it easier to spot when a widget has changed its own display context.

**Replay re-render reliability**
- Fixed a bug where replaying a recording a second time (without navigating away) could leave the widget in its previous internal state — edit mode, scroll position, tool selection, etc. Each replay run now forces a full re-injection of the widget HTML, so the widget always starts from a clean state.

### Fixes

- **Recording settle window**: the per-action safety timeout was incorrectly set to 30 seconds, causing the recorder to hang visibly after each interaction before moving on. Reduced to 5 seconds, matching the intended design.
- **Recording snapshot timing**: snapshots for recorded steps are now taken at whichever comes first — 150 ms after the action, or immediately when the next action arrives. This prevents a concurrent tool-call response from rewriting the widget iframe before the snapshot is captured.
